第4章続きシグナル発生ロジック

4.3 シグナル発生ロジック（矢印表示）

ネットスコア方式
買い矢印・売り矢印は「買いpt合計 - 売りpt合計」のネットスコアで判定する。
買い矢印: net_buy（= 買いpt - 売りpt）≥ 2 かつ リセット状態
売り矢印: net_sell（= 売りpt - 買いpt）≥ 2 かつ リセット状態

リセット条件

  • net_buy ≤ 0 → 買いリセット（次の買いシグナル発生可能）
  • net_sell ≤ 0 → 売りリセット（次の売りシグナル発生可能）
  • 売りシグナル発生時 → 買いリセットも同時にセット

スコア合算対象

スコース変数内容
バックエンドスコア_backendScores[k]APIから取得したルールベーススコア（買い/売り各pt）
チャートパターンpatScores[k]フロントエンドで自動検出（各2pt）
描画トレンドラインtlScores[k]ユーザー描画TLのスコア（上方+1pt/下抜け-2pt）

4.4 スコアバー表示

チャート右上に表示。カーソルを合わせたローソク足の合算スコアをリアルタイム表示。

  • 買い +Npt 緑色で表示
  • 売り -Npt 赤色で表示

第5章トレンドライン機能

5.1 描画フロー

  1. ヘッダーの「TL」ボタンをクリック → 描画モード開始（ボタン黄色、カーソルが十字）
  2. チャート上で第1点をクリック → ボタンが「待機中」表示に変わる
  3. 第2点にマウスを移動 → 破線プレビューがリアルタイム表示
  4. 第2点をクリック → ラインが確定、DBに自動保存（API: POST /trendlines）
  5. 描画モードは自動解除される
注意: TLボタンを再クリックすると描画モードをキャンセルできる。描画中にESCキーは未対応。

5.2 編集・削除

操作方法動作
ホバーハイライトライン近傍にマウスを移動ラインが黄色に強調表示
選択ライン近傍をクリックラインが赤に変わり選択状態
削除選択状態で中点の「×」をクリックDB削除（API: DELETE /trendlines/{id}）
全削除分析パネルの「全削除」ボタン当銘柄・時間足の全TL削除（API: DELETE /trendlines）
Y方向移動選択状態でドラッグ両端点を並行移動、マウスアップ時にDB更新

5.3 スコアへの影響（TLスコア計算）

各ローソク足に対してユーザー描画TLのスコアを計算（calcTLScores()）:

  • 終値がTLより上: +1pt（支持として機能）
  • 前日終値≥TL かつ 当日終値<TL（下抜け）: -2pt（ブレイクアウト下向き）

このTLスコアはバックエンドスコア・チャートパターンスコアと合算してシグナル判定に使用される。

6.1 損益サマリー（上部）

インポートされた全銘柄の損益を一覧表示。上部領域（デフォルト約70%）。

要素詳細
タイトル行「損益サマリー」+ 合計P&L金額（利益: グリーン、損失: レッド）
お気に入り星★でDBお気に入り登録、☆で未登録。クリックでトグル
銘柄名/コードクリックでチャートに銘柄切替
P&L金額利益: グリーン、損失: レッド
ウォッチタグウォッチリスト銘柄に「ウォッチ」グレータグ表示
部分保有タグ一部のみ売却済の場合「(一部保有中)」表示
選択行現在チャート表示中の銘柄を青背景でハイライト

P&L計算ロジック

  • 買い: 全約定の加重平均コスト計算
  • 売り: コスト基準 = 買コスト × (売数量 / 買数量)
  • P&L = 売上金 - コスト基準
  • 保有中は「保有中」タグのみ表示（P&L非表示）

リサイズハンドル

第8章CSVインポート

8.1 対応フォーマット

証券会社の約定履歴CSVファイルに対応。

列名（CSVヘッダー）データ型備考
約定日date「YYYY/MM/DD」または「YYYY-MM-DD」
銘柄コードstring数字4桁 or XXXX.T 形式
銘柄名string日本語銘柄名
売買区分string「買」「売」「買付」「売付」等
数量number取引株数
単価number約定単価（円）
手数料number任意
税金number任意
受渡金額number受渡金額（売却時の実現額）

8.2 処理仕様

  • エンコーディング: UTF-8 → Shift-JIS → UTF-8 (fallback) の順で自動判定
  • 複数ファイル: 同時に複数ファイルを選択可能（ドラッグ&ドロップ対応）
  • 集約: 同日・同銘柄・同売買方向は1レコードに集計（加重平均単価）
  • 重複除外: サーバー側 UNIQUE KEY で二重インポートを防止
  • 銘柄自動登録: 未登録銘柄はウォッチリストに自動追加
